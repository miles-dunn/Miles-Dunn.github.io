About

Not a resume in website form.

I wanted this site to feel more like a guided conversation than a list of credentials. So instead of trying to cram every class, tool, and bullet point into one page, I’m focusing on the work I’ve enjoyed making and the way I think while building it.

The common thread in my projects is that I like taking something messy — a vague idea, a clunky workflow, an annoying manual process — and turning it into something clearer and easier to use.

What I’m drawn to

Backend engineering AI integration Automation Clean UI Security-minded design

How I like to work

  • Start with the user problem first
  • Keep systems simple enough to reason about
  • Ship something real, then refine the edges
